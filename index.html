NAMED ENTITY EXTRACTOR

Information Retrieval and Extraction, IIIT Hyderabad




Motivation Behind this Project

Explore ways of finding and making use of named entities in web search.

Approach Adapted to Implement the project

Classification of article title as entity or non entity

Resective Tiitles of the wikipedia articles are used to assist in the classification of the articles into their respective domains based on whether or not the title is a named entity. This is achieved by checking the Capiitalization of the title.

Extracting meaningful data from XML dumps.

Most of the raw data in XML format eeds to be preprocessed to tackle the noise, generating useful data needed for extracting various Named entities. The implementation in the project has ensured the same, thus maintiing the qualty of the extracted named entitiies.

Extract Links and redirects.

This is mammothly responsible for entity linking and subsequent context derivation in Named Entity Extraction. Including this approach helps in refining the search a great deal.

Classification of entities based on Category membership

Various wikipedia cues have been extracted to dig the categories to which an extracted entity belongs. Wildcards in the wiki info boxes have been exploited to further judge the categorization of data

Extra Features!


Complex 3-Tier Posting list!

This reduces the primitive large order time taken to extract named entities and makes the extraction a lot faster

Entity-Linking

Improves the relevance of the search!

Word Sense Disambiguation

In cases wherein the same word may refer to multiple meanings, the project implements various approaches to filter the sense of the word on the basis of the users rest of the query.

Interactive Interface

The easy to use , user friendly UI makes it simpler to use the project.
